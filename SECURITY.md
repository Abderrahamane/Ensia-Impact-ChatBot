# Security Policy

## Supported Versions

This project is provided as-is for educational and portfolio usage. Security fixes are applied on the latest main branch.

## Reporting a Vulnerability

If you discover a vulnerability:

1. Do not open a public issue with exploit details.
2. Contact the maintainer privately with:
   - affected file/component
   - reproducible steps
   - expected impact
3. Allow time for patching before public disclosure.

## Secrets and Data Safety

- Never commit `.env` or API keys.
- Rotate any credential that was ever shared in screenshots or chat.
- Keep private Telegram exports out of public repositories.
- Use least-privilege API keys for cloud deployments.
